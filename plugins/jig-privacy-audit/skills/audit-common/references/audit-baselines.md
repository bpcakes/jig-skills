# Audit Baselines

Use these baselines to orient audit language. Verify current details from official sources when a user asks for citations, a compliance mapping, or a standards-specific assertion.

- OWASP ASVS: web application security verification controls.
- OWASP WSTG: web and API testing methodology.
- OWASP MASVS: mobile secure storage, cryptography, authentication, networking, platform interaction, code quality, resilience, and privacy checks.
- OWASP Cryptographic Storage Cheat Sheet: storage encryption and cryptographic storage anti-patterns.
- OWASP Password Storage Cheat Sheet: password hashing with modern slow password hashing algorithms and unique salts.
- OWASP Logging Cheat Sheet: sensitive data handling and protection of logs.
- OWASP Secrets Management Cheat Sheet: central storage, provisioning, auditing, rotation, and lifecycle of secrets.
- NIST SP 800-57: key management concepts and lifecycle guidance.
- NIST SSDF: secure software development practices.
- SLSA: supply-chain integrity and provenance framework.
- Sigstore: artifact signing and transparency-log-backed verification ecosystem.
- OpenSSF Scorecard: automated open-source project security-health heuristics.

These references provide audit framing. They do not by themselves prove that a product is secure, compliant, or zero knowledge.
